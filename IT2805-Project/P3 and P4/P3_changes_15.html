Changes from P2

Our plan was to create six HTML pages for content, with a common stylesheet, five javascript files for functionality, and three modularity files to avoid duplicating the same code. During the development process, we have made several changes to the project structure and the original style plan. All changes made are described below.

Changes to booking.html

We made a new javaScript file, ‘calculatePrice.js’, to calculate the price at the moment the user checks/unchecks the checkboxes for the desired service in booking.html. In the original plan we did not add the making of a seperate js file for calculating price.

Instead of displaying a text under the ‘Bekreft bestilling’ button, we chose to have a pop-up window which displays a confirmation of time and date of your appointment in a new window. The parent page will refresh and become empty. This will make it clearer that you have managed to complete a booking, and we avoid customers pressing the booking button twice.

We have added a heading to the booking.html page, this to automatically make it clear for the customer what the purpose of the page is.

We have added a dropdown window for choosing time for booking as well as the date. We chose the dropdown instead of input type = time, as it is easier for the stylist to set service times.

We changed the text of the pop-up to show time + date, instead of weekday + date.

Changes to pricelist.html

On pricelist, we originally had rounded sides in the box wrapping the price table. We made a decision to change it to a square box, as the other design elements have the square shape and we strive to have a consistent design.

The original plan was to have empty tables in pricelist.html, and an array with services and prices. When loading the page, we wanted to loop through the array, and print the content in the tables in pricelist.html. This should make the pricelist easy to maintain, with few code changes when updating the content.
While implementing this, we decided it would be nicer to make the tables in javascript too, so everything would depend on the array. As a result, pricelist.html only contains a div-box, and prices.js will print a number of tables with content. The number of tables created depends on the content in the array is. This makes it easier to update the pricelist-site in the future, because you only need to change the array, and then the tables will be remade based on the new array.

In order to improve pricelist.html and make it more exclusive, we added the image carousel between the footer and the price table.

Changes to header.js

Navbar is kind of transparent.

Originally, titles in the navbar were chosen to have a pink color when hovering over them, to make the title stick out. As we changed the navbar to be more transparent, the pink color became more invisible. Therefore, we added an underline in the navbar, as it makes it more clear to the users when they are hovering over a title.

We renamed “Hjemmeside” to “Hjem”, as home is a more fitting name on such a link.

Changes to dropdown.js

Originally planned as an own file, containing the functionality for the dropdown menu when hovering “Våre salonger”. When implementing the navbar it made much more sense to merge this functionality into header.js. As a result, dropdown.js was removed.

Changes to about.html

From the get go, we wanted to have picture of Frida to scale horizontally with the text about her, but we realized this would be a problem when the width resolution gets too small. With interest of scalability we moved the picture of Frida beneath the text when the width of the users’ medium reaches 1159px.

Changes to collage.part

After writing the HTML code for the collage.part, we quickly realized it contained too many unique elements to be a modularity file. So instead of the original collage.part, we decided to make the collages as normal HTML-elements included in the homepage.html and about.html

Changes to the Image Carousel

Originally, we planned to have an image carousel included in homepage.html, with a carousel.js file for its functionality. As stated earlier, we decided to have an image carousel on the pricelist.html site as well, and in order to make this implementation smooth, we did several changes:

The carousel.js script file that tells how the carousel should behave, was renamed to rotateCarousel.js

We made a modularity file to avoid copying a lot of HTML code from homepage.html to pricelist.html. Now, the image carousel is created in a carousel.js file located in the modularity folder, and then we import it into homepage.html and pricelist.html.

When the carousel became an own file, we also created an own stylesheet, carousel.css, that also is imported it into homepage.html and pricelist.html.

CSS Changes

We didn’t take scalability into account during our planning, however with the interest of it we have made some changes to each site to scale to a certain degree. Therefore we decided to make a burger meny when the browser width is less than 800px. As a result, our site is now pretty mobile friendly.

Many small design changes have been made during the development process. We had a design template/design guide, but quickly realised that some font-sizes had to be resized in order to make the website look well. For instance we ended up using px instead of pt, as we want pixel accuracy. When it came to the various headings described in the design template, some of these were slightly changed since we saw that they needed to be altered.

When it came to visual hierarchy related to the content, we made some changes to the font thickness, sizing and white space. For instance, we made the undertitles bold in the Salon pages. Ensuring popper use of the Gestalt principles for white space and the elements makes a better visual. This is also related to usability as it it easier to scan a page and look for the correct content.

General Changes from P2

In the interest of getting a structured and intuitive code, with few repeated code lines, we originally decided that the header and footer should have their own script, with some common ids to be used across all sites. This should ensure that we could change our sites accordingly when needed.
When implementing the website, we realised that the same should apply for the tab title and favicon, so we made two more scripts: tilte.js and favicon.js. This change ensures that all tabs have the same title and icon. In the earlier stages of the project, favicon was forgotten. As a result, we also have added a new image to our project, favicon.png.
Since we use a JavaScript file to add attributes to the favicon HTML-element, an error occurs when validating the HTML-code. We have tested the site on different browsers on several devices, and we cannot see that this error has a huge impact on our site. We could add all link attributes in every HTML-file, but we would rather change the content in one JavaScript file, than update every file, but get rid of the error.

After consulting with our student teaching assistant, we came to the conclusion that having one big stylesheet was messy, and long to read. So we decided to create a css file for each respective site. These sites would only have local style changes, while the main style.css had the more general, and common style across the sites.

Upon further work with the project, we decided to update our folder structure, pictures of the employees was moved a level up since it wasn’t intuitive to have these pictures grouped together with /hair folder. Since Frida, the founder was still an employee of Øya, we decided to move her picture together with the rest of the employees. We have also added an own folder for pictures used in homepage/about. We decided to scale these pictures on the same resolution and apply the same style to each of them, instead of resizing each individually, this also future proofed additional pictures, as long as they have the same resolution.
